За единый стандарт коммендировая в Joostina принят формат PhpDocumentor ( http://ru.wikipedia.org/wiki/PhpDocumentor или более толково http://it-giki.ru/post/206.html , более наглядно - http://www.slideshare.net/roadhump/phpdocumentor ).

Каждый PHP файл должен содержать единообразную "шапку", со всей необходимой информаций касающейся данного файла.
/**
 * Шапка типичного php файл
 * С комментариями на несколько строк, столько сколько понадобится для ясного представления о структуре и лицензировании файла
 * 
 * Файл принадлежит компоненту "Мегакомп"
 * @author JoostinaTeam <info@joostina.ru>
 * @version 1.0
 * @package MegaCom
 * @subpackage model
 */


Каждый класс необходимо комментировать аналигичным способом:
/**
 * МегаКласс
 *
 * Класс работы с классами, класный и интересный
 * @package MegaCom
 * @subpackage classes
 */

Классы относящиеся к моделям должны принадлежать субпакету joosDbModel [ или же для этих целей использовать category ? - надо проштудировать всё и посмотреть что получится правильнее ]:
/**
 * СуперТабличныйМегаКласс
 *
 * Класс работы с моделью базы данных СуперТаблица
 * @package MegaCom
 * @subpackage joosDbModel
 * @category joosDbModel
 */
